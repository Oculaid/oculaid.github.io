Software Application-level Development

Sensor fusion, feedback and low power mode development complete!

Posted by Jaskaran Basson on March 9, 2021

Sensor Fusion

Originally, our plan was to use a Kalman Filter to estimate and predict object locations through time, based on input from multiple sensors. However, after extensive sensor characterization and testing, we determined that the sensor variances were quite small. This means that we do not need any object estimation or tracking, as object detection using the sensor measurements at every time step is sufficient to detect obstacles. Consequently, the implemented algorithm uses the closest sensor measurement as the obstacle distance in the forward direction (first-person POV). To determine whether the obstacle is to the left, centre, or right in the horizontal direction, the mounting locations of the sensors are used. For example, if only the left sensor saw the object, it is to the left, whereas if the centre or both left and right sensors saw the object, it is in the centre. Using the horizontal and forward location of the detected object, we appropriately actuate the feedback.

Feedback

The feedback is relayed to the wearer in the form of vibrations at the ends of the glasses behind the ears. Depending on the horizontal location of the detected object, the left, right or both vibration motors are actuated. Further, based on how distant the object is in the forward direction, the intensity of the vibrations are varied. The mapping which determines the distance to intensity ratio was determined after extensive user testing from multiple wearers.

Next, software support for the two switches was implemented. To offer the most flexibility and comfort to the wearer, the OculAid consists of two switches that allow the user to configure the level of the detection and feedback sensitivity. The detection sensitivity ranges from [High, Low, Off], which detect objects 2m away, 1m away, or turn the glasses off, respectively. The feedback sensitivity ranges from [High, Med, Low], which configure the intensity of the vibrations. This will allow the glasses to serve the user with optimal comfort and configuration to their preference!

Low Power Mode

Low power mode has been implemented! Using the IMU, the glasses can now successfully detect when they have been sitting idle for a set amount of time and put themselves in low power mode to save battery. This set amount of time, or period, is easily configurable but has been set to 30s for this prototype for demonstration purposes. Upon detecting movement, the glasses will immediately wake up and exit low power mode, ready to serve the user!
